The product: golflessons.be

Golf lesson booking platform for pros and students

  • Pros subscribe, configure availability, receive payments
  • Students browse pros, book lessons, get coached
  • Sister site of silverswing.golf — shared identity layer
  • Three languages from day one: EN, NL, FR

Next.js 16

Full-stack React framework by Vercel

  • Server Components — HTML rendered on the server, zero JS shipped
  • App Router — file-based routing with layouts, loading states, error boundaries
  • Server Actions — form submissions that run server-side code directly
  • Turbopack — Rust-powered dev server, instant hot reload
  • API routes, middleware, streaming — one framework does it all

TypeScript

JavaScript with types — catches bugs before runtime

  • Strict mode — no implicit any, null safety everywhere
  • Database schema generates TypeScript types (Drizzle)
  • API responses are type-checked end to end
  • Refactoring with confidence — rename and the compiler finds every reference

Tailwind CSS + shadcn/ui

Utility-first styling + accessible component library

  • Tailwind — style directly in markup, no separate CSS files
  • Design tokens as CSS variables — one change updates everything
  • shadcn/ui — copy-paste components you own (not a dependency)
  • Built on Radix primitives — accessible, keyboard-navigable, composable

Drizzle ORM + Neon

Type-safe database access + serverless PostgreSQL

  • Drizzle — SQL-like syntax, full TypeScript inference, zero overhead
  • Schema-as-code — migrations generated from TypeScript definitions
  • Neon — PostgreSQL that scales to zero, branches like git
  • Serverless-friendly — HTTP connection pooling, no persistent connections needed

Vercel

Deploy on push — preview every branch, ship to production on merge

  • Git-integrated — every push gets its own URL
  • Edge network — fast globally, no infrastructure to manage
  • Fluid Compute — serverless functions that reuse instances
  • Blob storage, environment variables, domains — all managed
  • Zero config for Next.js — just push and it works

Stripe Connect

Marketplace payments — students pay, pros receive

  • Express accounts — pros onboard in minutes with hosted forms
  • Split payments — lesson fee goes to pro, platform takes 2.5%
  • Automatic SEPA payouts to pro's bank account
  • Webhooks handle the lifecycle — subscription, payment, payout events
  • Platform subscription (annual) + per-lesson commission model

Request flow

Browser → Vercel Edge → Middleware (auth gate, role check)

↓

Server Components → Drizzle ORM → Neon Postgres

↓

Server Actions → Stripe / Gmail / Blob

No separate API server. Everything runs on Vercel Functions.

Backup & Restore

  • Daily cron dumps the whole DB to Vercel Blob as versioned JSON
  • /dev/backups — view, download, restore, delete
  • Test script: pnpm test:backup validates round-trip
  • 90-day retention, automatic cleanup alongside notifications

Never lose data. Restore any point in time.

Event logging

  • Dedicated events table with indexes for time-range queries
  • logEvent() helper — fire-and-forget, never breaks the caller
  • Instrumented: bookings, logins, pushes, OAuth, backups
  • /dev/logs — filter, search, color-coded levels, auto-refresh
  • Runs forever on Postgres — millions of events, sub-second queries

One page, three log views

Events → our app's business events (bookings, pushes, auth)

Runtime → Vercel function logs via Vercel REST API

Sentry → uncaught errors with stack traces & breadcrumbs

Three systems, one tab switch. No context jumping.

Error tracking with Sentry

  • Installed via Vercel Marketplace — unified billing, SSO, one click
  • Captures every uncaught error (server + client + edge)
  • Stack traces with source maps, user attribution, breadcrumbs
  • Custom error.tsx branded with our palette
  • Webhook → internal notifications → phone push via ntfy

A bug in production pages me within seconds.

The alert chain

Error in production

↓ captured by Sentry

↓ Sentry fires issue:created webhook

↓ /api/sentry/webhook verifies HMAC signature

↓ createNotification() + direct ntfy push

→ Phone vibrates within 5 seconds

Health checks

  • /api/health — JSON endpoint checking DB, Stripe, Blob, env vars, Sentry
  • Returns 200 "status":"ok" or 503 "status":"degraded" with details
  • /dev/health — branded dashboard with per-check latency
  • ?deep=1 — expensive checks (Stripe/Blob) for manual/periodic use

External monitoring

Self-hosted Uptime Kuma on our Hetzner node

  • Polls /api/health every 60 seconds from outside Vercel
  • Catches what internal checks can't: DNS, TLS expiry, domain issues, Vercel itself
  • Four monitors: prod health, preview health, deep check, plain home
  • Same ntfy push channel → one notification path for everything
  • Zero recurring cost — runs alongside existing services on Telmio

Full observability stack

  • Events table — business events
  • Vercel logs — raw stdout/stderr
  • Sentry — errors & stack traces
  • Uptime Kuma — external polling
  • ntfy — phone push channel
  • /dev/* — one dashboard

Production-grade monitoring for a pre-launch product. All built in one afternoon.
